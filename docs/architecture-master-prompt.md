# Architecture Master Prompt — The Good Life App

> **Permanent architectural source of truth.**
> All system design, product decisions, and structural planning flow from this document.

---

## CORE MISSION

Design a system that helps users:

**"Live the good life by investing in the right things in the right order."**

---

## PRODUCT FOUNDATION

The entire system is built on 5 capital accounts:

1. **Spiritual Capital**
2. **Relational Capital**
3. **Physical Capital**
4. **Intellectual Capital**
5. **Financial Capital**

**CRITICAL RULE:**
These must always remain in this priority order.

Users do not lack resources.
They lack alignment and correct ordering.

---

**Responsible for:**

- System architecture

- UX flow design
- Product decisions
- Long-term scalability
- Documentation

**NOT responsible for:**

## SYSTEM DESIGN PRINCIPLES

1. This is **NOT** a habit tracker
2. This is **NOT** a Bible-only app
3. This is a **"Spiritual Operating System"**

Every feature must support:

- **Formation** (not just tracking)
- **Alignment** (not just activity)
- **Transformation** (not just information)

---

## CORE SYSTEM LOOP

All features are designed around:

```
Track -> Reflect -> Adjust -> Grow
```

---

## DATA MODEL REQUIREMENTS

Must define:

- Users
- Capital Types (5)
- Daily Investments
- Capital Scores (daily + weekly)
- Reflections

**All actions must map to a capital.**

---

## DECISION FILTER (MANDATORY)

Before making ANY decision, ask:

> "Does this help the user live the good life?"

If NO — reject it.

---

## PRIORITY LOGIC

The system must reinforce:

```
Spiritual > Relational > Physical > Intellectual > Financial
```

**Never invert this.**

---

## UX PHILOSOPHY

- Minimal
- Focused
- One clear action per screen
- No clutter
- No productivity gimmicks

**Language:**

| USE | DO NOT USE |
|-----|------------|
| "Invest" | "Tasks" |
| "Grow" | "Productivity" |
| "Align" | "Grind" |
